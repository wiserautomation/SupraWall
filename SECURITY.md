# Security Policy

## Supported Versions

Only the latest version of SupraWall is currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within SupraWall, please send an e-mail to security@supra-wall.com. 

### Coordinated Disclosure Policy
- We will acknowledge your email within **48 hours**.
- We will provide a status update and a target fix timeline within **7 days**.
- We request that you do not disclose the vulnerability publicly until a fix is released.
- Upon successful remediation, we will credit you in our [Security Advisory](#) and, where applicable, assign a CVE.

## Vulnerability Disclosure Timeline
1. **Report:** Researcher emails security@supra-wall.com.
2. **Analysis:** Internal triage within 48h.
3. **Fix:** Engineering works on a patch.
4. **Validation:** Researcher verifies the fix in a private staging environment.
5. **Release:** Patch is released to all supported versions.
6. **Publication:** CVE or Security Advisory is published.
